# Руководство по участию в проекте

Спасибо за интерес к проекту "Telegram Bar Assistant"! Мы приветствуем любой вклад в развитие бота.

## 🚀 Как начать

### Настройка окружения разработки

1. **Fork репозиторий** на GitHub
2. **Клонируйте** ваш fork:
```bash
git clone https://github.com/yourusername/telegram-bar-assistant.git
cd telegram-bar-assistant
```

3. **Создайте виртуальное окружение:**
```bash
python -m venv venv
source venv/bin/activate  # Linux/Mac
# или
venv\Scripts\activate  # Windows
```

4. **Установите зависимости:**
```bash
pip install -r requirements.txt
```

5. **Настройте переменные окружения:**
```bash
cp .env.example .env
# Добавьте ваши API ключи в .env
```

6. **Создайте ветку для разработки:**
```bash
git checkout -b feature/your-feature-name
```

## 🔧 Стандарты разработки

### Стиль кода
- Следуйте **PEP 8** для Python кода
- Используйте **type hints** где это возможно
- Добавляйте **docstrings** для функций и классов
- Максимальная длина строки: **88 символов** (Black formatter)

### Структура коммитов
Используйте [Conventional Commits](https://www.conventionalcommits.org/):

```
<type>[optional scope]: <description>

[optional body]

[optional footer(s)]
```

**Типы коммитов:**
- `feat:` - новая функция
- `fix:` - исправление бага
- `docs:` - изменения в документации
- `style:` - форматирование кода
- `refactor:` - рефакторинг без изменения функциональности
- `test:` - добавление или изменение тестов
- `chore:` - обновление зависимостей, конфигурации

**Примеры:**
```bash
feat(rag): add support for cocktail name declensions
fix(stt): handle OpenRouter API timeout errors
docs: update installation instructions
```

### Тестирование
- Добавляйте тесты для новой функциональности
- Убедитесь, что все существующие тесты проходят
- Запустите тесты перед отправкой PR:
```bash
pytest tests/
```

### Логирование
- Используйте модуль `logging` вместо `print()`
- Добавляйте информативные сообщения для отладки
- Используйте соответствующие уровни логирования:
  - `DEBUG` - детальная информация для отладки
  - `INFO` - общая информация о работе
  - `WARNING` - предупреждения о потенциальных проблемах
  - `ERROR` - ошибки, которые не останавливают работу
  - `CRITICAL` - критические ошибки

## 📋 Процесс Pull Request

1. **Убедитесь, что ваша ветка актуальна:**
```bash
git checkout main
git pull upstream main
git checkout your-feature-branch
git rebase main
```

2. **Запустите тесты и проверки:**
```bash
pytest tests/
python -m flake8 .
python -m black --check .
```

3. **Создайте Pull Request:**
- Используйте описательное название
- Опишите изменения в деталях
- Ссылайтесь на связанные Issues
- Добавьте скриншоты для UI изменений

4. **Шаблон описания PR:**
```markdown
## Описание
Краткое описание изменений.

## Тип изменения
- [ ] Исправление бага
- [ ] Новая функция
- [ ] Критическое изменение
- [ ] Обновление документации

## Тестирование
- [ ] Тесты проходят локально
- [ ] Добавлены новые тесты
- [ ] Протестировано вручную

## Чеклист
- [ ] Код следует стилю проекта
- [ ] Добавлены комментарии для сложной логики
- [ ] Обновлена документация
- [ ] Изменения не ломают существующую функциональность
```

## 🏗️ Архитектура проекта

### Основные компоненты
- **handlers/** - обработчики Telegram сообщений
- **services/** - бизнес-логика и API клиенты
- **rag/** - система поиска по базе знаний
- **data/** - база знаний и кэш

### Принципы разработки
- **Разделение ответственности** - каждый модуль имеет четкую задачу
- **Асинхронность** - используйте `async/await` для I/O операций
- **Обработка ошибок** - graceful degradation при недоступности API
- **Конфигурируемость** - все настройки через переменные окружения

## 🍹 Добавление новых коктейлей

### Формат данных
Создайте файл в `data/documents/` со следующей структурой:

```
[CHUNK]
id: COCKTAIL_XXX_NAME
tags: cocktail, category, spirit
keywords: name, name, русское название, как сделать name, рецепт name
text: Категория: category. Состав (мл): ингредиент1 количество; ингредиент2 количество. Метод: способ приготовления. Подача: тип стакана, гарнир. Вкус: описание вкуса.
```

### Рекомендации
- Используйте стандартные категории: `spirit-forward`, `sour`, `fizz`, `long`
- Указывайте точные пропорции в миллилитрах
- Добавляйте русские и английские варианты названий в keywords
- Описывайте подачу детально для правильной генерации изображений

## 🎨 Улучшение генерации изображений

### Промпты для коктейлей
Редактируйте `services/router.py`:
- `_create_cocktail_image_prompt()` - основная логика
- `_extract_color_info()` - определение цвета по ингредиентам
- Добавляйте специальные случаи для сложных коктейлей

### Кэширование
- Изображения автоматически кэшируются в `data/images/`
- Используйте `/cache_stats` для мониторинга
- Система автоматически очищает дубликаты
