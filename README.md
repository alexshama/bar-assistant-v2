# Барный ассистент v2

Telegram-бот для барменов и команд бара: рецепты, справка по напиткам, рекомендации по выбору ингредиентов, голосовые запросы, анализ изображений и генерация иллюстраций коктейлей.

## Что умеет бот

- Текстовые ответы по коктейлям, крепкому алкоголю, пиву и подаче.
- Локальный поиск по базе знаний из `data/documents/`.
- Рекомендации по выбору напитка или ингредиента для коктейля.
- Генерация изображений коктейлей через OpenRouter с fallback на OpenAI.
- Голосовые запросы: STT и голосовые ответы.
- Анализ фотографий напитков через vision-модель.
- Админ-команды для кэша и переиндексации базы знаний.

## Архитектура v2

- `handlers/` отвечает только за Telegram-события.
- `services/` содержит бизнес-логику, API-клиенты и общие утилиты.
- `rag/` реализует облегченный RAG-слой на JSON-индексе.
- `data/documents/` хранит исходные текстовые документы и чанки.

Важно: текущий RAG в проекте не использует полноценную векторную БД. В v2 это легковесный JSON-индекс с нормализацией запросов и scoring по `chunk_id`, `tags`, `keywords` и тексту документов.

## Локальный запуск

1. Создайте виртуальное окружение:

```bash
python -m venv venv
```

2. Активируйте его:

```bash
# Linux / macOS
source venv/bin/activate

# Windows PowerShell
venv\Scripts\Activate.ps1
```

3. Установите зависимости:

```bash
pip install -r requirements.txt
```

При желании для более надежной конвертации голосовых сообщений через `pydub` установите `ffmpeg`:

```bash
# Ubuntu / Debian
sudo apt install -y ffmpeg
```

4. Создайте `.env`:

```bash
# Linux / macOS
cp .env.example .env

# Windows PowerShell
Copy-Item .env.example .env
```

5. Заполните обязательные переменные:

```env
TELEGRAM_BOT_TOKEN=...
OPENAI_API_KEY=...
OPENROUTER_API_KEY=...
ADMIN_USER_IDS=123456789
```

6. Запустите бота:

```bash
python main.py
```

## Конфигурация `.env`

Обязательные:

- `TELEGRAM_BOT_TOKEN`
- `OPENAI_API_KEY`
- `OPENROUTER_API_KEY`

Полезные дополнительные:

- `ADMIN_USER_IDS` — список Telegram user id через запятую.
- `LOG_LEVEL` — `INFO`, `WARNING`, `DEBUG`.
- `REQUEST_TIMEOUT_SECONDS` — таймаут внешних API.
- `AUTO_BUILD_INDEX_ON_STARTUP` — автоматически строить индекс, если его нет.
- `OPENROUTER_IMAGE_MODEL` — модель для генерации изображений через OpenRouter.
- `IMAGE_CACHE_PATH`, `USER_SETTINGS_PATH`, `DOCUMENTS_PATH`, `CHROMA_DB_PATH` — пути к runtime-данным.

Все настройки и пути валидируются в `config.py`.

## Команды

- `/start` — приветствие и краткое описание возможностей.
- `/help` — примеры запросов.
- `/mode` — переключение между `подробно` и `кратко`.
- `/voice on` / `/voice off` — включить или выключить голосовые ответы.
- `/reset` — сброс контекста и пользовательских настроек.
- `/stats` — runtime-статистика процесса.
- `/reindex` — ручная переиндексация базы знаний.
- `/cache_stats` — статистика кэша изображений.
- `/clear_cache` — очистка кэша изображений, только для админов из `ADMIN_USER_IDS`.

## База знаний

Документы лежат в `data/documents/`.

Чтобы обновить базу знаний:

1. Добавьте или отредактируйте `.txt`-файлы в `data/documents/`.
2. Выполните `/reindex` в боте или удалите `data/chroma_db/simple_index.json`.
3. Перезапустите приложение, если хотите, чтобы индекс пересобрался автоматически.

## Надежность v2

В проекте усилены:

- валидация env-переменных;
- кроссплатформенные пути через `pathlib`;
- startup bootstrap;
- логирование без вывода секретов;
- fallback для image/STT/TTS сценариев;
- retry и таймауты для OpenAI и OpenRouter;
- сохранение пользовательских настроек в `data/user_settings.json`;
- вынос повторяющейся логики отправки ответов из хендлеров.

## Ограничения

- Генерация изображений зависит от внешних API и может быть дорогой.
- Конвертация голоса в WAV остается best-effort: если `pydub` или `ffmpeg` недоступны, бот старается работать с исходным OGG.
- Пользовательские настройки сохраняются локально в JSON и подходят для одного инстанса бота; для горизонтального масштабирования понадобится БД.
